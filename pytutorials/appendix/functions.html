Basic Functions

Function which takes one argument

import math # Used for next_square()
''' Example:

    >>> say("Yo quiero Taco Bell")
    Yo quiero Taco Bell
'''

def say(message):
    print(message)
    

''' Return the next square
    Examples:
    
    >>> next_square(4) = 9
    >>> next_square(25) = 36        
'''

def next_square(x):
    # We call a number a "square" if its square root is an integer
    # The if statement checks if x is an integer
    if float(int(math.sqrt(x))) != math.sqrt(x):
        raise ValueError('{0} is not a square'.format(x))
    else:
        return (math.sqrt(x) + 1)**2

Function with an optional/default argument

The function below has an optional argument. If a value for the argument is not provided when calling the function, a default value ("Yo quiero Taco Bell") will be used.

def speak(message="Yo quiero Taco Bell"):
    print(message)

Examples

>>> speak()
Yo quiero Taco Bell

>>> speak("Hello world!")
Hello world!

Combining Optional and Required Arguments

def introduce(town,name="Steve"):
    print("Hi, my name is {0} and I am from {1}.".format(name,town))

Functions with Multiple Arguments

For the introduce() example above, these following calls are equivalent:

  1. introduce("Oakland","Vincent")
  2. introduce(town="Oakland",name="Vincent")
  3. introduce("Oakland",name="Vincent")
  4. introduce(name="Vincent",town="Oakland")

You might be asking, what are the rules Python uses such that these four different function calls are treated the same?

  • Arguments can be explicitly named (4) or not (1)
    • If they are not explicitly named, then Python assumes you are specifying arguments in the order in which they are defined
  • Arguments which are explicitly named can be called in any order
  • All keyword arguments should be placed after non-keyword arguments
    • This call will produce an error:
      introduce(name="Vincent",Oakland)

Tip: Explicitly name your arguments when possible

This is a matter of taste, but there are two very good reasons why we should name our arguments:

  • You might not (probably won't) remember the arguments of a particular function from code you last edited over a week ago

Functions with Arbitrary Arguments

''' Return the product of all of its arguments:

    Example:
    >>> product(1,2,3)
    6
    
    >>> product(10,10,0)
    0

'''
def product(*terms):
    product = 1
    for i in terms:
        product *= i
    return product

Functions with arbitrary keyword arguments

''' Example:

    >>> drive_thru(option='cheeseburger')
    I would like a cheeseburger please.

    >>> drive_thru(option='cheeseburger',side='fries')
    I would like a cheeseburger with fries please.
    
    >>> drive_thru(option='cheeseburger',side='fries',drink='Coke')
    I would like a cheeseburger with fries and a Coke please.
    
'''
def drive_thru(**kwargs):
    order = 'I would like a '
    if ('option' in kwargs):
        order += kwargs['option']
    if ('side' in kwargs):
        order += ' with ' + kwargs['side']
    if ('drink' in kwargs):
        order += ' and a ' + kwargs['drink']
    order += ' please.'
    
    print(order)

Functions with both arbitrary arguments and keyword arguments

''' Examples:

    >>> print_quote("If you're not first you're last.",author="Eleanor Roosevelt")
    "If you're not first you're last." - Eleanor Roosevelt
    
    >>> print_quote("If you're not first you're last.","I'M VERY ANGRY",author="Eleanor Roosevelt")
    "IF YOU'RE NOT FIRST YOU'RE LAST." - Eleanor Roosevelt

'''

# Note: Calling this function without any arguments will produce an error unlike the
# product() or drive_thru() examples
def print_quote(message,*args,**kwargs):
    # Surround quote with ""
    message = '"{0}"'.format(message)
    
    if ("I'M VERY ANGRY" in args):
        message = message.upper()
    if ('author' in kwargs):
        message += ' - {0}'.format(kwargs['author'])
        
    print(message)

Recursive Functions

Recursive functions are functions which call themselves. You should be careful when constructing recursive functions and ensure that they eventually terminate, or else they may potentially repeat forever.

# Compute n!
# See: http://stackoverflow.com/questions/5136447/function-for-factorial-in-python
def factorial(n):
    if (n == 0):
        return 1
    else:
        return n * factorial(n-1)

Don't ever do this

This function has no termination conditions and will continue forever until either Python crashes or you hit Control + C.

def dont_do_this():
    print("Don't ever do this")
    dont_do_this()

Closure

# Functions within functions can pull variables from their parent scope, e.g.
# inner() has access to all of the variables within outer()'s scope

def outer():
    message = "Hello world!"
    
    def inner():
        print(message)
    
    inner()
