Composite Data Types

In computer science, a "composite data type" is basically a fancy word for describing data types that can contain other data types, like numbers, strings, lists, and so on. In Python, composite data types like lists, tuples, and dictionaries can contain virtually any other Python object.

A Brief Introduction to Lists, Tuples, and Dictionaries

These are Python's basic composite data types. Here, there will be a few examples of values that can be stored in them and them some more specific details on their use.

Lists

Python's lists are like... lists. They can store any number of items of varying type. Lists can be appended to, i.e. have items added to, at any time. Furthermore, its contents are mutable. That is, they can be edited.

Declaring a List

To declare a list simply assign a comma separated list, enclosed in square brackets, to a variable.

>>> x = [1,2,3]
>>> x
[1, 2, 3]
>>> x = ["a","b","c"]
>>> x
['a', 'b', 'c']
>>>

Tuples

Tuples are ordered pairs of objects. They resemble lists in many ways, but are noticeably different in that:

  • They are immutable, i.e. their contents cannot be changed
  • They are fixed length, which means we can't append to them like lists
There's a peculiar debate in the Python community over where lists are appropriate and where tuples are, but for now, just understand what they are and what their limitations are.

Declaring a Tuple

To declare a tuple simply assign a comma separated list (normal English meaning), enclosed in parentheses or nothing at all, to a variable.

>>> x = 1,2,3
>>> x
(1, 2, 3)
>>> x = (1,2,3)
>>> x
(1, 2, 3)
>>>

Dictionaries

Dictionaries in Python resemble dictionaries in real life in that they pair items with associated values. The items that are paired to values are called keys.

For example, if we wanted a dictionary where we associated fruits with their colors, we could declare it like this:

fruits = {
	'apple': 'red',
	'orange': 'orange',
	'blueberry': 'blue',
	'strawberry': 'red'
	}

In general, the syntax for declaring a dictionary is:

dictionary = {key: value, key: value, ... , key: value}

Putting every key-value pair on a separate line like I did in the previous example is not necessary, but is considered generally good etiquette as it is easier to read.

Operations on Composite Data Types

All composite data types in Python have a set of operations that can be performed on them, though the specifics of those operations may differ.

Indexing: Accessing Individual Items

Every individual item inside a container has an index, which is basically a word or number which we can use to retrieve it.

List and Tuple Indexing

Individual items in lists and tuples can be retrieved by using a number corresponding to their position in the list or tuple.

This is going to be really confusing but... the first item in a container has the index 0, not 1. The index 1 refers to the second item. This seemingly strange convention is not unique to Python, and can be found in many other programming languages. There is actually a logical explanation for this, which we won't get in to.

Syntax: variable_name[index]

An example of list and tuple indexing

>>> x = [1,2,3]
>>> x[0]
1
>>> x[1]
2
>>> x = 1,2,3
>>> x[0]
1
>>> x[1]
2
>>> x[2]
3
>>>
The last item of a list and tuple can be retrieved with the index -1 
>>> x[-1]
3

Dictionary Indexing

We can access individual items in dictionaries with similar syntax to lists, except instead of integers, we use the key associated with the value we want to get.

Syntax: dictionary_name[key]

Example

Using the fruits example from above:

>>> fruits["apple"]
'red'
>>> fruits["orange"]
'orange'
>>>

Using dict.keys()

We can get a list (sort-of) of the indices (keys) of a dictionary using dictionary_name.keys():

>>> fruits.keys()
dict_keys(['apple', 'orange', 'strawberry', 'blueberry'])
>>> fruits.keys()[0]
Traceback (most recent call last):
  File "", line 1, in 
    fruits.keys()[0]
TypeError: 'dict_keys' object does not support indexing
>>> list(fruits.keys())
['apple', 'orange', 'strawberry', 'blueberry']
>>> list(fruits.keys())[0]
'apple'
>>>

As you may have already inferred, the object returned by dict.keys() is not a list, but can be easily converted to one.

Dictionaries

Syntax: dictionary_name[key] = value

Example

Using our fruits example again, we'll add another entry into our dictionary: 

>>> fruits
{'apple': 'red', 'orange': 'orange', 'strawberry': 'red', 'blueberry': 'blue'}
>>> fruits["watermelon"] = "green"
>>> fruits
{'apple': 'red', 'orange': 'orange', 'strawberry': 'red', 'watermelon': 'green', 'blueberry': 'blue'}
>>>
