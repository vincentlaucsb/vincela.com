Here, we'll cover some operations we can perform on numbers. Most of them will be mathematical.

Basic Math Operations

Importing Modules

In addition to some basic built-in functions, Python also includes dozens of built-in modules. A module is simply a collection of functions and objects (which you will learn about later). Most modules are created for a specific purpose, e.g. statistical analysis, or web development, so their functions will be oriented around that purpose as well. Because these modules can literally include hundreds of different functions, they are not automatically loaded. To load these modules, we can use the import statement.

Namespaces

But before importing a module, we'll go over a hasty and brief explanation of what namespaces are and why they exist. Most of us have encountered real life situations where we have multiple people that share the same name working in the same space. As a result, we have to invent a new way to referring to these people, e.g. "Nick 1" and "Nick 2", or "Nicholas", "Nick", and "Nikki."

Likewise, multiple Python modules can share similar function names. We could imagine multiple Python math and statistics modules all containing an add() function which performs completely different things depending on the module. Namespaces are Python's solution to the problem of similar names.

Importing the Math Module

import math

This statement imports all of the functions of the module named math into the namespace also called math. To use the functions inside of the math namespace, we use the following syntax: 

math.function_name

Listing the functions in the namespace

By using the dir() function, we can list all of the contents of the math namespace.

['__doc__', '__loader__', '__name__', '__package__', '__spec__', 'acos', 'acosh', 'asin', 'asinh', 'atan', 'atan2', 'atanh', 'ceil', 'copysign', 'cos', 'cosh', 'degrees', 'e', 'erf', 'erfc', 'exp', 'expm1', 'fabs', 'factorial', 'floor', 'fmod', 'frexp', 'fsum', 'gamma', 'gcd', 'hypot', 'inf', 'isclose', 'isfinite', 'isinf', 'isnan', 'ldexp', 'lgamma', 'log', 'log10', 'log1p', 'log2', 'modf', 'nan', 'pi', 'pow', 'radians', 'sin', 'sinh', 'sqrt', 'tan', 'tanh', 'trunc']

While it is obvious what many of these functions do just by looking at their names (which is a good programming practice), we can find more detailed documentation here from the official Python documentation.

Do not worry too much about the functions surrounded in __underscores__. These functions are intended by use for Python, and not the user (although in more advanced later lessons, we might tamper with some of these __functions__).

Some Examples

Basic Math

>>> math.sqrt(4)
2.0

Mathematical Constants

>>> math.e
2.718281828459045
>>> math.pi
3.141592653589793

Trigonometric Functions

>>> math.radians(180)
3.141592653589793
>>> math.sin(0)
0.0
>>> math.cos(0)
1.0
>>> math.cos(math.pi)
-1.0
>>> math.cos(-math.pi)
-1.0
>>> math.cos(2*math.pi)
1.0

Logarithms

Here, the second (optional) argument is the base of the logarithm. If the base is not specified, Python returns the natural logarithm (log with base e).

>>> math.log(16,2)
4.0
>>> math.log(100,10)
2.0

Importing Specific Functions

We can also import specific functions from the math module by typing:

from math import function_name

However, unlike with import math, here our functions get imported into the global namespace, i.e. the namespace that contains our familar print() function and all the variables we have been working with (so far). To call something in the global namespace, we do not need to prefix it with anything—we just call it by its name. Now, the obvious danger in this is that we may accidentally import two functions with the same name from different modules into the global namespace.

Importing an entire module into the global namespace

Lastly, you may occasionally see a wildcard import, where an entire module gets imported into the global namespace.

from math import *

However, for the name conflict issue mentioned earlier, this is generally not recommended.
