What is a Boolean statement?

From Wikipedia:

In computer science, a Boolean expression is an expression in a programming language that produces a Boolean value when evaluated, i.e. one of true or false.
Boolean statements are named after George Boole, who thought of the idea of representing false statements as 0s, and true statements as 1s. This simple idea is now what underpins all of modern computing.

Why do we use them?

As you may have seen earlier, a lot of times when writing functions, we want to do one thing if a certain condition is true, but another thing is it is false.

We evaluate "Boolean statements" in our everyday lives as well. For example, before walking out the door, you might evaluate something like, "If it there are zombies outside, I will grab my gun. Otherwise, I'll leave it home." This statement can be translated into Python-like pseudocode (fake code) like this: 

proposition = "There are zombies"
if proposition is True:
    get_weapon("rifle")
    get_weapon("flamethrower")
else:
    do_nothing()
    live_normally()

In the previous example, the Boolean statement was the proposition "there are zombies." This proposition has two values, True or False. The if branch of our program executes if the proposition is True. The else branch executes if it is False.

Also notice the capitalization of True and False. This is because Python has these keywords reserved to refer to the potential values of Boolean statements. Anytime you want to refer to Boolean values, you should always use True or False capitalized in that way. Likewise, you should never use True or False as variable, function, etc. names.

A Cool Trick with Boolean Statements

In Python, empty strings, lists, dicts, and other container objects evaluate to False.

>>> bool("Just gonna stand there and watch me burn")
True
>>> bool([1,2,3,4,5])
True
>>> bool({"staples":500,"pens":10,"notebooks":5,"post-its":200})
True
>>> bool("")
False
>>> bool([])
False
>>> bool({})
False

Why does this matter?

There are many times when we're coding where we will need to check if, for example, a list is empty or not. In Python, we have a neater, shorter way to check if something contains items or not.
